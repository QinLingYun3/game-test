# 麻将连连看游戏开发提示词

请帮我用 HTML/CSS/JS（原生，不依赖外部库）开发一个 H5 版本的**麻将主题连连看**游戏，要求如下：

## 1. 视觉风格

- **麻将主题**：每个卡片使用麻将牌面图案（如筒、条、万、字牌、花牌），可用 Unicode 符号或 emoji 代替，也可用简易图形 + 文字表示（例如 🀙、🀚、🀛 或 “1筒”、“2条”、“東”）
- **牌面设计**：每个卡片有浅色或白色背景，带圆角，轻微阴影，类似真实麻将牌的外观
- **排列**：麻将牌呈网格排列，行列之间留有均匀间距
- **适配**：支持 PC 端（鼠标点击）和移动端（触摸点击），响应式布局

## 2. 核心玩法（经典连连看规则）

- **网格尺寸**：建议 6×8 或 8×8，确保每种牌面数量为偶数
- **消除条件**：
  - 点击两张相同的麻将牌
  - 两张牌之间的连接路径最多只能拐 **两个弯**（即经典的连连看路径规则）
  - 路径只能经过空白区域（已被消除的位置）
- **牌面种类**：至少 8~10 种不同的麻将牌
- **开局**：随机打乱所有牌的位置，确保每种牌数量为偶数，并且存在至少一个可解的解（建议使用预置可解布局或洗牌后验证）
- **无解检测**：当没有可消除的对时，自动洗牌（或者提示“无可用对，重新洗牌”，并手动/自动触发洗牌）

## 3. 交互与反馈

- **选中效果**：点击第一张牌时高亮显示（边框发光或缩放效果），点击第二张牌时尝试消除
- **消除成功**：
  - 两张牌同时消失，置为空位
  - 播放简单动画（淡出或缩小消失）
  - 增加分数
- **消除失败**：
  - 如果不满足相同牌面或路径规则，取消选中，无消除
  - 提示音效可选（不做强制，但可提示文字反馈）
- **胜利条件**：所有麻将牌被消除 → 显示 "You Win!" 并提供「新游戏」按钮
- **洗牌功能**：提供「洗牌」按钮，重新排列当前未被消除的牌（保持牌面组合不变）

## 4. 分数与计时（可选但推荐）

- **分数**：每次消除增加 10 分，实时显示
- **最高分**：存储到 `localStorage` 并显示
- **计时器**（可选）：从游戏开始计时，完成时显示耗时

## 5. 路径算法要求

- 实现经典的连连看路径检测函数：
  - 同一条直线（0 拐点）
  - 一个拐点（两条直线组成 L 型）
  - 两个拐点（Z 或 U 型，两条平行线）
- 路径检测时只考虑空位（已消除的格子）

## 6. 技术实现

- 纯 HTML/CSS/JS，不依赖第三方库
- 使用 Grid 或 Flex 布局生成牌桌
- 状态管理清晰（二维数组存储牌面与是否消除）
- 所有代码封装在一个 `.html` 文件中，复制即可运行

## 7. 额外体验优化

- 选中不同步时自动取消选中（如点第一张后再点另一张不同牌时，第二张成为新的选中）
- 消除后自动取消高亮
- 洗牌按钮带有防抖，避免频繁洗牌
- 移动端触摸友好（点击区域足够大）
