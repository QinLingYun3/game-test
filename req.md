# H5 2048 风格游戏开发提示词

请帮我用 HTML/CSS/JS（原生，不依赖外部库）开发一个 H5 版本的 2048 风格游戏，要求如下：

## 1. 视觉风格

- 类似原版 2048，但配色更现代柔和（例如浅色背景、卡片轻微圆角、阴影）
- 支持 PC 鼠标操作

## 2. 核心玩法

- **网格**：4x4 网格，初始随机生成两个数字（2 或 4）
- **操作方式**：
  - PC 端：支持键盘方向键
- **合并规则**：与原版 2048 一致，相同数字碰撞时合并为倍数，每次移动后随机空位生成一个新数字（2 或 4）
- **游戏结束检测**：无空位且无可合并相邻块时结束，显示 "Game Over" 弹窗/提示，并提供「重新开始」按钮
- **分数系统**：实时显示当前分数（每次合并时累加合并后的数值）

## 3. 动画与交互细节

- **动画效果**：每次移动时，卡片移动、合并、新卡片出现都应有平滑过渡动画（CSS transition 或简单 JS 动画）
- **游戏结束反馈**：伴随震动感（移动端可选），至少弹窗 + 重试按钮
- **撤销功能**（加分项，可选）

## 4. 技术实现

- 纯 HTML/CSS/JS，不依赖第三方库
- 布局使用 Flex 或 Grid，适配不同手机分辨率（设置 viewport）
- 代码结构清晰，包含必要的注释
- 所有逻辑封装在一个文件中，可直接复制为 `.html` 文件在浏览器打开运行

## 5. 额外要求

- 最高分存储到 `localStorage` 并显示
- 增加「新游戏」按钮
- 动画流畅，无卡顿

请生成完整代码。
